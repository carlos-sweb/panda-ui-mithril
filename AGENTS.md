# panda-ui-mithril

Mithril.js component library styled with Panda CSS. daisyUI-compatible class naming.
Icons via lucide-mithril.

## Commands

- Dev server: `npm run dev`
- Build library: `npm run build:lib`
- Build playground: `npm run build`
- Panda codegen: `npm run codegen`
- Type check: `npm run typecheck`

## Architecture

```
src/components/    -> One folder per component (index.js + index.d.ts + *.styles.js)
src/styled-system/ -> Auto-generated by Panda CSS (never edit manually)
playground/        -> Demo site with Vite dev server
src/index.js       -> Barrel file re-exporting all components
```

## Code Style

- Mithril components use object literal pattern: `{ view(vnode) { ... } }`
- Never use `m()` without importing `m` from 'mithril'
- Styles always via className, never inline style objects
- Use `cva()` for single-element components, `sva()` for multi-slot components
- Import Panda utilities from '../styled-system/css' or '../styled-system/patterns'
- Icon components from 'lucide-mithril': `import { IconName } from 'lucide-mithril'`
- One component per file. Named exports only, no default exports.
- Type definitions in adjacent .d.ts files, exported from src/index.d.ts

## Component Pattern

Each component folder contains:

- `index.js` -> Mithril component (view function)
- `index.d.ts` -> TypeScript declarations
- `*.styles.js` -> cva/sva recipe definitions

## Naming Convention

All CSS class names follow daisyUI convention:
- Lowercase, hyphen-separated: `btn-primary`, `card-body`, `alert-soft`
- Component base class first: `btn`, `card`, `alert`
- Modifiers: `{component}-{modifier}` -> `btn-primary`, `card-border`
- Parts: `{component}-{part}` -> `card-body`, `modal-box`
- Colors: `{component}-{color}` -> `btn-primary`, `badge-error`
- Sizes: `{component}-{size}` -> `btn-lg`, `input-sm`

## Boundaries

- Never edit src/styled-system/ (auto-generated by panda codegen)
- Never commit node_modules/
- panda.config.ts controls token theme - changes affect all components
- All class names follow daisyUI convention: lowercase, hyphen-separated
